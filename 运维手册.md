## 1. 目录结构

```

├── stock_screen.py            # 统一入口：--top N / --months N / --no-history / --maintain

│   ├── __init__.py
│   ├── data_sources.py        # 腾讯行情/月线 + 新浪大盘指数
│   ├── screener.py            # 评分引擎 + 批量筛选 + 信号标签
│   ├── pool_manager.py        # Core≤20 / Radar≤50，淘汰+补入+晋升+评分持久化
│   └── reporter.py            # 日报告输出 + 历史归档 + 旧报告清理

│   ├── history.json           # 每日 Top5 快照（90天滚动上限）
│   └── pools/                 # 首次运行自动创建

│       ├── radar_pool.json    # 格式同上
│       └── score_history.json # {symbol: [{date,total_score,drawdown,months,...}]} 90天滚动

```

## 2. CLI 参数

```bash
python3 stock_screen.py                          # 默认 Top 10，12 个月周期
python3 stock_screen.py --top 20 --months 60     # Top 20，5 年长周期
python3 stock_screen.py --months 120              # 10 年极长周期
python3 stock_screen.py --months 1               # 1 个月短周期，振幅阈值极紧
python3 stock_screen.py --no-history             # 不写 history.json（调试用）
python3 stock_screen.py --maintain               # 周度全市场扫描+淘汰+补入+晋升
python3 stock_screen.py --maintain --months 36   # 维护模式用 3 年周期
```

| 参数 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| `--top` | 10 | 1~N | 输出 Top N |
| `--months` | 12 | 1~120 | 评测周期（月），振幅阈值按 √(months/12) 自适应缩放。上限 120 = 10年（与数据源一致） |
| `--no-history` | — | — | 跳过 `data/history.json` 写入 |
| `--maintain` | — | — | 全市场扫描 + 淘汰 + 补入 + 晋升 |

## 3. 调用链路

### 日常筛选 (`stock_screen.py` / `--months N`)
```
daily_run()
  ├─ run_screener(top_n=N, months=M)          # 对 Core+Radar 评分排序
  │    ← get_watchlist_for_screener()        #   合并 Core+Radar，去重
  │    → analyze_stock(sym, name, months)    #   逐票调用腾讯接口
  │         → get_tx_quote()                 #     实时行情
  │         → get_tx_monthly(120)            #     月线(前复权，降序排序)
  │         → tech_score(..., months)        #     振幅阈值按 sqrt(months/12) 自适应
  ├─ record_scores(results)                  # 写入 pools/score_history.json
  ├─ clean_stale_history()                   # 清理不在池中的僵尸评分
  └─ generate_report(results, months=M)
       ├─ get_sina_market_overview()         # 大盘指数
       ├─ 写入 output/YYYYMMDD_report.json + .md
       ├─ 追加 data/history.json（保留90天）
       └─ _cleanup_old_reports()             # 删除30天前的旧报告
```

### 周度维护 (`stock_screen.py --maintain`)
```
maintain_pools()
  ├─ get_full_market_symbols()               # HS300 ∪ ZZ500 去重
  ├─ 用已有 pool 数据补齐 watchlist 的 name/sector
  ├─ run_screener(watchlist=全市场, months=M) # 过滤 total_score=0 的无效数据
  ├─ record_scores(valid_candidates)          # 持久化（带 months 字段）
  └─ run_cleanup_cycle(candidates, months=M)
       ├─ get_broken_out(core, months=M)     # 只看同 months 条目，回撤连续5天> -10% → demote
       ├─ get_bottom_stagnant(radar, months=M) # 只看同 months 条目，连续30天 score<5.0 → remove
       ├─ feed_radar(candidates)             # Top 5 补入 Radar
       ├─ promote_radar_to_core()            # 高分晋升 Core
       ├─ trim_radar()                       # 超50只裁掉最低分
       └─ clean_stale_history()
```

## 4. 评分体系

**综合分 = 价值分 × 0.6 + 技术分 × 0.4**（0~10）

### 价值分 `value_score(pb, pe)`
| 维度 | 权重 | 满分线 | 零分线 | 衰减 |
|------|------|--------|--------|------|
| PB | 60% | ≤0.3 | >1.0 | 0.3→0.7 线性(10→0), 0.7→1.0 线性(5→0) |
| PE | 40% | ≤5 | >50 | 5→20 线性(10→5), 20→50 线性(5→0) |

- PB=0 合法（破净股），`pb is None` 判断缺失
- PE≤0 直接返回 0

### 技术分 `tech_score(drawdown, amplitude, ma_div, months)`
| 维度 | 满分 | 条件 | 梯度 |
|------|------|------|------|
| 回撤深度 | 4分 | 较60月高点跌幅>50% | 4/3/2/1 对应 >50/>40/>30/>20 |
| 横盘程度 | 3分 | 近N月振幅<20%×√(N/12) | 3/2/1 对应 <20/<35/<50 × scale |
| 均线粘合 | 3分 | MA5/10/20差异<5% | 3/2/1 对应 <0.05/<0.10/<0.20 |

- `scale = max(0.2, √(months/12))`，12月阈值 20/35/50，60月自动缩放为 45/78/112
- 信号标签（`get_signal_tag`）的振幅阈值与 `tech_score` 同步缩放

## 5. 数据源

| 数据 | 来源 | 接口 | 解析器 | 零值策略 |
|------|------|------|--------|----------|
| 实时行情 | 腾讯证券 | `qt.gtimg.cn` | `sf()` | 0=无数据→None |
| PB 字段 | 腾讯证券 | 同上 | `sf_pb()` | 0=合法(破净) |
| 月线数据 | 腾讯证券 | `web.ifzq.gtimg.cn` | `sf_kline()` | ≥0.001=合法 |
| 大盘指数 | 新浪财经 | `hq.sinajs.cn` | `sf()`+`sf_safe()` | price用sf，pct/chg用sf_safe(0=合法) |

- 月线使用前复权(qfq)，`get_tx_monthly` 返回前按日期**降序排序**，`[:N]` 取最近 N 个月
- 新浪大盘接口字段以**逗号**分隔（非 `~`）
- 所有 HTTP 调用均有 try/except 网络异常兜底

## 6. 池管理

| 池 | 容量 | 文件 |
|----|------|------|
| Core | ≤20 | `core_pool.json` |
| Radar | ≤50 | `radar_pool.json` |

### 淘汰规则
| 条件 | 函数 | 逻辑 | 动作 |
|------|------|------|------|
| 形态破坏 | `get_broken_out(months)` | 同 months 条目回撤连续5天 > -10% | Core→Radar |
| 长期垫底 | `get_bottom_stagnant(months)` | 同 months 条目最长连续30天 score<5.0 | Radar→删除 |
| 行业平衡 | `enforce_sector_cap()` | 单一 parent_sector >25% Core | 拒绝该行业新人 |

淘汰依赖 `score_history.json`，需 `record_scores` 积累数据（约需30天）。**不同 `--months` 评分完全隔离**，各 months 值独立统计连续天数，互不污染。

### months 隔离机制
- `record_scores` 写入时记录 `months` 字段
- `get_bottom_stagnant` / `get_broken_out` 过滤 `e['months'] == months`
- 旧历史无 months 字段 → `e.get('months', 12)` 归为默认 12
- 切换 months 等价于该配置**冷启动**

## 7. 数据防呆

| 机制 | 位置 | 说明 |
|------|------|------|
| `history.json` 90天滚动 | `reporter.py` | 每次追加后保留最近90条 |
| `score_history.json` 90天滚动 | `pool_manager.py` | 每票保留最近90条 |
| `output/` 30天清理 | `reporter.py` | `generate_report` 时触发 |
| 僵尸评分清理 | `pool_manager.py` | `daily_run` 和 `maintain` 时执行 |
| months 隔离 | `pool_manager.py` | 淘汰只统计同 months 条目 |
| 月线数据排序 | `data_sources.py` | `get_tx_monthly` 返回前降序排序 |
| 高/低校验 | `screener.py` | 逐月检查 high≥low，反转则交换 |
| 振幅防护 | `screener.py` | ≤0 或 >500 时跳过评分 |
| name 兜底 | `screener.py` | watchlist name 为空时用 API 返回的 name |
| sector 兜底 | `stock_screen.py` | 全市场扫描时用已有 pool 补齐 |
| 网络异常兜底 | `data_sources.py` | 全部 HTTP 函数 try/except |
| None 安全格式化 | `reporter.py` | Markdown 输出全部 get+or 取值 |

## 9. 踩坑清单

### 月线数据方向
腾讯 API 实际返回**旧→新**，`get_tx_monthly` 已内置降序排序。`[:N]` 取最近 N 个月。

### 新浪大盘分隔符
指数接口用 **`,`** 分隔字段，个股接口用 `~`。`get_sina_market_overview` 已修正为 `split(',')`。

### 数值解析器选择
| 场景 | 用 | 误用后果 |
|------|----|----------|
| 实时行情价/量/市值 | `sf()` | — |
| PB | `sf_pb()` | 用 `sf()` 则 PB=0→None |
| K 线 close/high/low | `sf_kline()` | 用 `sf()` 则 0 值低点被丢弃→振幅爆炸 |
| 涨跌幅/涨跌额 | `sf_safe()` | 用 `sf()` 则平盘日 0→None |

### 代码前缀规则
| 前缀 | 交易所 | 代码 |
|------|--------|------|
| `sh` | 上海 | 60xxxx, 688xxx |
| `sz` | 深圳 | 00xxxx, 30xxxx, 002xxx |

### 模块导入规范
- 入口: `sys.path.insert(0, 'modules')` + `from modules.xxx import yyy`
- 子模块: `sys.path.insert(0, os.path.dirname(__file__))` + `from xxx import yyy`
- 禁止相对导入 `from .xxx import yyy`

### `--no-history`
跳过 `data/history.json` 写入，不影响 `record_scores` 到 `score_history.json`。

## 11. FAQ

| 现象 | 原因 | 解决 |
|------|------|------|
| 股票无数据 | 交易所前缀错误 | 对照 sh/sz 代码范围纠正 |
| 技术分异常 | 月线切片方向反了 | 检查用 `[:N]` 非 `[-N:]` |
| 振幅异常大/负 | 低点数据被错误过滤 | 已修复（排序+阈值+校验） |
| 大盘概况为空 | 新浪分隔符或网络问题 | 已修复（逗号分隔+异常兜底） |
| Core 池股票失踪 | 形态破坏触发降级 | 查看 score_history drawdown |
| 淘汰不生效 | score_history 无数据 | `record_scores` 已注入，数据需积累 |
| 切换 months 后淘汰不触发 | months 隔离机制冷启动 | 等该 months 值积累 30 天即可 |
| sector 为"其他" | 新扫描票无历史数据 | 正常，入池后可修正 |
| `data/pools/` 不存在 | 首次运行 | 自动创建 |
| 报告堆积 | 30天自动清理 | 已内置 |
